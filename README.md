# Программа, составляющая конспект
Программа, состовляющая конспект по предоставленному материалу. 

У программы имеется 2 режима: режим классического конспектирования и режим конспектирования в формате вопросов для самопроверки. В режиме классического конспектирования на выходе программа составит конспект в свободной форме, тогда как в режиме конспектирования в формате вопросов для самопроверки программа составит нумерованный список из вопросов по предоставленному материалу и ответов к ним, также взятых из предоставленного материала. Формат записи полученного списка вопросов [описывается ниже](#формат-файлов-со-списками-вопросов).

## Использование
1. Убедитесь, что в вашей операционной системе установлен python версии *>= 3.0.0*, и путь к интерпретатору содержится в переменной окружения *PATH*.

2. Скачайте zip-архив со [cтраницы программы в github](https://github.com/vdm-17/summarizing_utility).

3. Распакуйте содержимое архива по пути, который не содержит кириллицу или спец. символы.

4. Установите необходимые переменные окружения в соответсвие с [инструкцией ниже](#установка-переменных-окружения).

5. Отредактируйте файл *config.ini* в соответствие с [инструкцией ниже](#конфигурация-программы).

7. Установите необходимы зависимости при помощи команды:  ```pip install -r requirements.txt```.

6. Перейдите в каталог программы и введите команду: ```python main.py ПУТЬ_К_ФАЙЛУ_С_МАТЕРИАЛОМ_ДЛЯ_КОНСПЕКТА_В_СИСТЕМЕ ПУТЬ_К_ФАЙЛУ_В_КОТОРЫЙ_НЕОБХОДИМО_СОХРАНИТЬ_КОНСПЕКТ --mode РЕЖИМ_КОНСПЕКТИРОВАНИЯ --details УРОВЕНЬ_ПОДРОБНОСТИ_КОНСПЕКТА```.

## Параметры запуска

* Флаг *-m* или *--mode* устанавливает режим работы программы. Для установки классического режима используется значение *-m classic*, которое используется по умолчанию. Для установки режима конспектирования в формате вопросов для самопроверки используется значение *-m questions*.

* Флаг *-d* или *--details* устанавливает уровень подробности конспектирования. Для установки низкой подробности используется значение *-d low*. Для установки средней подробности используется значение *-d medium*, которое используется по умолчанию. Для установки высокой подробности используется значение *-d high*. Для установки максимальной подробности используется значение *-d xhigh*.

## Формат записи полученного списка с вопросами
Простейший файл со списком вопросов выглядит так:
```
# Вопросы для самопроверки
1. Вопрос. ||Ответ на вопрос.|| [Источник.](Ссылка на исходный файл с предоставленным материалом в системе) Страницы: n1, n2. Фрагмент в тексте: ||Фрагмент в тексте, из которого был взят ответ на вопрос.||
```

## Установка переменных окружения

Создайте файл *.env* в основном каталоге программы и отредактируйте его следующим образом:
```
APP_ENV='ТЕКУЩЕЕ_ОКРУЖЕНИЕ'
OPENAI_API_KEY='ВАШ_OPENAI_API_КЛЮЧ'
```
где:

* *APP_ENV* (опционально) - текущее окружение программы. Может принимать значение *development* при разработке.

* *OPENAI_API_KEY* (обязательно) - ваш OpenAI API ключ. Подходят лишь официальные ключи OpenAI.

## Конфигурация программы
1. Откройте файл *config.ini*.

2. В разделе *\[DEFAULT\]* отредактируйте значение ключа *SUMMARIZING_AGENT_OPENAI_MODEL*, заменив его на необходимую модель OpenAI. По умолчанию используется значение *gpt-5.2*.

3. В разделе *\[DEFAULT\]* отредактируйте значение ключа *SUMMARIZING_AGENT_REASONING_EFFORT*, заменив его на необходимое значение усилиия обдумывания модели OpenAI. По умолчанию используется значение *high*.

4. В разделе *\[DEFAULT\]* отредактируйте значение ключа *SUMMARIZING_AGENT_VERBOSITY*, заменив его на необходимое значение разговорчивости модели OpenAI. По умолчанию используется значение *high*.
